# Phase 3 — UI Design Contract

> Visual and interaction contract for Core Game Loop phase. Players roll dice, move tokens around a 40-tile board, and advance turns with ongoing drains applied automatically. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Host Board Display Contract

### Layout
- **Board representation:** Circular/square track layout (40-tile looping board)
- **Track style:** Visual path showing all 40 tiles as connected spaces (SVG or CSS grid acceptable)
- **Player representation:** Colored dot per player (unique color), positioned on current tile
- **Current turn highlight:** Active player's dot has larger size (1.5× scale), glow effect (box-shadow), or border

### Turn History Sidebar
- **Position:** Right side of board (on large screens) or below board (on mobile <768px)
- **Height:** 250px max (scrollable if >10 turns)
- **Content:** Last 5 turns displayed (oldest at bottom, newest at top)
- **Item format:** "[Player Name] rolled [N] → [Tile Name]" on single line, left-aligned
- **Font:** 14px (0.875rem), monospace for numbers, text color #aaa
- **Row spacing:** 8px vertical padding per item, #16213e background for contrast

### Tile Labels
- **Display:** Show tile name abbreviation (4–8 chars) on hover OR as small badge on each tile
- **Font:** 12px (0.75rem), weight 600, color #aaa
- **Abbreviations (Phase 3 defined tiles only):**
  - Payday → "PAYDAY"
  - Prison → "PRISON"
  - Park Bench → "BENCH"
  - Hospital → "HOSP"
  - Apartment → "APART"
  - House → "HOUSE"
  - Career entrances (10 total) → "[CAREER_NAME]" (e.g., "TECH", "FINANCE")
  - Opportunity tiles → "OPP" (with career initials on hover)
  - TBD tiles (14 remaining) → "TBD" (placeholder until Phase 4–8 finalize)

### Turn Counter Display
- **Position:** Top center or top right of board
- **Format:** "Turn [N] of [Max Turns]" or just "Turn [N]" (exact format: Claude's discretion, but include turn number)
- **Font:** 16px (1rem), weight 600, color #f0c040
- **Visibility:** Always visible; updates on every turn advancement

## Player Screen Display Contract (Roll Phase)

### Roll Dice Button
- **Text:** "Roll Dice" (uppercase optional)
- **Size:** 48px (vertical) × 60% width (max 300px)
- **Font:** 20px (1.2rem), weight 700, color #1a1a2e (on #4ade80 background)
- **State — Enabled:** Background #4ade80, cursor pointer, hover state #22c55e (darker green)
- **State — Disabled:** Background #4ade80, opacity 0.35, cursor not-allowed
- **Position:** Center of screen, prominent
- **Visibility guard:** Only visible when `turnPhase === WAITING_FOR_ROLL` AND `currentTurnPlayer === this player`

### Current Player Indicator
- **Text:** "[Player Name]'s Turn" (if it's this player) OR "Waiting for [Player Name]..." (if it's another player's turn)
- **Font:** 18px (1.125rem), weight 600, color #eee (or #f0c040 if it's their turn)
- **Position:** Above "Roll Dice" button
- **Visibility:** Always visible during game phase
- **Alignment:** Center

### Drain Notifications (Phase 3 applies drains, display is required)
- **Location:** Pop-up or inline message above roll button (not a separate notification center)
- **Format:** "Drains Applied:" followed by list (each on own line):
  - "-$2,000 Marriage" (if married)
  - "-$1,000 × [N] Kids" (e.g., "-$2,000 × 2 Kids" if 2 kids)
  - "-$1,000 Student Loan" (if has loans)
- **Font:** 14px (0.875rem), color #f87171 (red for negative)
- **Duration:** Display for 2–3 seconds, then fade out (CSS transition or timer)
- **Dismissable:** Optional — can auto-hide or require tap to dismiss
- **Timing:** Emit `drains-applied` event before roll button appears; client shows notification, then button enables

### Current Money Display (recommended, not required)
- **Text:** "$[Amount]" (e.g., "$50,000")
- **Font:** 16px (1rem), monospace, weight 600, color #eee
- **Position:** Below roll button or in status section
- **Updates:** On every money change (drains, tile effects) via `character-update` or gameState broadcast

## Host Game Section Layout

### Overall Structure
- **Header:** Turn counter (top center) + current player name (top center-right)
- **Main content:** Circular board (60–80% of width, centered)
- **Sidebar:** Turn history panel (right side on desktop, below on mobile)
- **Footer:** Game phase status (optional: "Turn phases: WAITING_FOR_ROLL / MID_ROLL / LANDED / WAITING_FOR_NEXT_TURN")

### Responsive Breakpoints
- **Desktop (1024px+):** Side-by-side layout (board left 70%, history right 30%)
- **Tablet (768px–1023px):** Board full width, history below (height 200px, scrollable)
- **Mobile (<768px):** Single column, board full width, history collapsed (expandable, height 150px when expanded)

## Socket Events & Client-Server Messaging (UI Implications)

### Inbound Events (Client receives)
| Event | Payload | UI Action |
|-------|---------|-----------|
| `move-token` | `{roll, fromPosition, toPosition, steps}` | Animate token movement (500ms or step-by-step) |
| `drains-applied` | `{playerId, deductions: [{type, amount}], newMoney}` | Show drain notification, update money display |
| `nextTurn` | `{currentTurnIndex, currentPlayer, currentPlayerName}` | Update turn indicator, enable/disable roll button, add to turn history |
| `tile-landed` | `{playerId, tileIndex, tileType}` | Log to turn history, highlight tile on board (optional flash/glow) |
| `gameState` (periodic broadcast) | Full room state | Sync turn order, current player, board positions |

### Outbound Events (Client sends)
| Event | Payload | When |
|-------|---------|------|
| `roll-dice` | {} | Player taps "Roll Dice" button during WAITING_FOR_ROLL |
| `play-experience-card` | `{cardId}` | Player selects Experience card instead of rolling (Phase 3 stub; Phase 7 implements) |

## Animation & Feedback Specifications

### Token Movement Animation
- **Duration:** 500ms total per roll (or step-by-step if multiple tiles traversed)
- **Easing:** Linear or ease-out (smooth, natural movement)
- **Method:** CSS transition or requestAnimationFrame
- **Behavior:** Start position → end position; server does NOT wait for animation to complete before emitting tile-landed

### Drain Notification Animation
- **Entrance:** Fade in + slide down (200ms), yellow/red color
- **Display:** 2–3 seconds static
- **Exit:** Fade out (300ms)
- **CSS:** Use `opacity`, `transform: translateY()`, `transition: all 300ms ease-out`

### Button State Transitions
- **Hover (enabled):** Background color shift (#4ade80 → #22c55e), 200ms transition
- **Click (enabled):** Optional pulse or brief scale-down (100ms) for tactile feedback
- **Disabled state:** No hover effect, opacity 0.35

### Tile Highlight (on landing)
- **Style:** Subtle glow or border highlight on the tile just landed on (optional, Phase 3 minimum viable)
- **Duration:** 1–2 seconds
- **Method:** CSS box-shadow or background-color transition

## Accessibility & Responsive Design

### Touch Targets (Mobile)
- **Minimum button size:** 44px × 44px for "Roll Dice" button (meets WCAG 2.1 Level AAA standard)
- **Spacing:** 16px minimum margin between interactive elements
- **Form inputs:** Min 44px height (observed in Phase 2: 12px padding = 36px+, acceptable; can increase to 44px if needed)

### Color Contrast
- **Text on backgrounds:** #eee on #1a1a2e (contrast ratio >7:1, passes WCAG AAA)
- **Button text:** #1a1a2e on #4ade80 (contrast ratio >7:1, passes WCAG AAA)
- **Error text:** #f87171 on #1a1a2e (check ratio ~4:1, acceptable but aim for >7:1 if re-darkening background)

### Responsive Layout
- **Desktop (1024px+):** Multi-column (board + history side-by-side)
- **Tablet (768px–1023px):** Single column, stacked sections, 80% board width
- **Mobile (<768px):** Full-width single column, 90% board width, collapsed history section
- **Small mobile (<480px):** Consider shrinking board or hiding turn history until expanded

### Screen Reader Support (baseline, not required for Phase 3)
- **Board:** Use `aria-label` on tiles (e.g., "PAYDAY tile, position 0")
- **Button:** "Roll Dice" label is semantic (use `<button>` element)
- **Turn indicator:** Announce "[Player Name]'s Turn" via ARIA live region (optional Phase 3)

## Design Notes & Rationale

### Board Display Approach
Phase 3 does not require character portraits (deferred to Phase 9). Minimal viable implementation: colored dots representing players, positioned on a circular/square track showing all 40 tiles. Tile effects are not displayed in Phase 3 (stubs only); Phase 4–8 will add effect overlays and animations.

### Turn History Format
Chosen: "[Player Name] rolled [N] → [Tile Name]" for clarity and consistency with ROADMAP success criteria "last 5-10 turns shown as `[PlayerName] rolled [N] → [TileName]`."

### No Drain Floor Dialog
Money drains automatically at turn start and floor at $0. No popup or confirmation—just broadcast the deduction and update display. Tile effects (which may also spend money) come in Phase 4+.

### Experience Cards (Phase 3 Stub)
`play-experience-card` socket event is stubbed in Phase 3 (no handler implemented yet). UI does not expose this button until Phase 7 when cards are actually granted on career completion. Phase 3 focuses on 2d6/1d6 rolling only.

### Tile Abbreviations
14 remaining board tiles are TBD (Phase 4–8 finalize exact positions and types). Phase 3 labels them as "TBD" on the board. Phase 4+ will update abbreviations as tile types are locked.
