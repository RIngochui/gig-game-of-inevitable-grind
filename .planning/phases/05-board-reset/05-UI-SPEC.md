## Component Inventory

**Player Screen (game section):**
1. **Turn Indicator** — Text element, center-aligned, color #aaa, 18px weight 600, "Your turn" or "Waiting for [Player Name]'s turn"

4. **Stat Grid** — 2 rows × 3 columns, each stat shows: [label (left-aligned, 12px #aaa uppercase)] / [value (18px weight 600 #eee monospace)] — replaces #money-display div
   - Row 1: Money | Fame | Happiness

5. **Active Tile Instruction** — Text block below stat grid, 14px color #aaa, "Currently on: [Tile Name]" + instruction paragraph

2. **Current Player Display** — 18px weight 600 color #eee, "[Player Name]'s turn"
3. **Board Track** — 11×11 grid, perimeter tiles only (40 total), existing structure maintained
4. **Tile** — Each tile:
   - Background #1a1a2e, secondary #16213e when current
   - Border 1px #333, 2px #f0c040 when current
   - Content: abbreviated tile name (12px monospace #aaa) + player dots below

6. **Current Tile Highlight** — Gold border (#f0c040) on active player's current tile

## Interaction Contract

**Player Screen Turn Flow:**
1. Turn starts: `advanceTurn` socket event fires, updates turn-indicator
2. Player lands on tile: `tileResolved` socket event fires, updates active-tile-instruction
3. Player sees current stat values (refreshed via `getFullState` broadcast)
4. If Player's turn: Roll Dice button enabled, click triggers `roll-dice` event
5. Roll completes: stat values update, new tile instruction appears

**Host Screen Turn Flow:**
1. Turn advances: `advanceTurn` broadcasts
2. turn-counter increments, current-player-display updates to next player
3. Board tiles update: player dots move to new positions, current-tile shows gold highlight
4. Turn history adds new entry: "[Player Name]: rolled [X, Y], moved to [Tile Name]"

**Tooltip Display (Host Board):**
- On ::after pseudo-element hover
- Content: tile instruction text from GAME-DESIGN.md
- Position: fixed relative to tile, prevent overflow with max-width 200px
- Dismiss: mouse leave

**Stat Grid Updates:**
- Real-time: listen to `getFullState` broadcasts and re-render stat values
- No animation (instant display), but color flash on change is optional (out of Phase 5 scope)

**Pre-populated from:** CONTEXT.md (D-05 specifies player screen instruction display updated each turn), Phase 3 existing socket event patterns (advanceTurn, tileResolved are already implemented).

## Implementation Notes

**Player Screen Stat Grid (replacing #money-display):**
- Replace `<div id="money-display">` with a grid container
- Grid structure: `display: grid; grid-template-columns: 1fr 1fr 1fr; grid-template-rows: auto auto; gap: 16px;`
- Each stat cell: flex column, label (12px #aaa uppercase), value (18px weight 600 #eee monospace)
- Values updated via `getFullState` socket listener, identical to existing money-display pattern

**Host Board Tile Tooltips:**
- Add `::after` pseudo-element to .tile class
- Content: `attr(data-instruction)` (populate via JS from tile instruction map)
- Positioning: absolute, left 50% transform translateX(-50%), top 110%, white-space nowrap, background #16213e, border 1px #f0c040, padding 4px 8px, z-index 10, border-radius 4px
- Visibility: opacity 0 by default, opacity 1 on .tile:hover
- Transition: opacity 200ms ease-out

**Player Screen Active Tile Instruction:**
- Add new div after stat grid: `<div id="active-tile-instruction">` 
- Content: `<p id="tile-name-display">Currently on: [Tile Name]</p><p id="tile-instruction-text">[Instruction]</p>`
- Update on `tileResolved` event or socket broadcast that includes current tile
- Styles: 14px color #aaa, monospace for tile name, sans-serif for instruction text, padding 16px, background #16213e, border-radius 8px, margin-top 16px

**Win Condition Check (server-side, no UI required but shown here for context):**
- Life Total = Fame + Happiness + floor(Cash / 10,000)
- Win check: Life Total ≥ 60 AND player's secret formula satisfied
- Runs after every stat change (handled in backend, not UI)

**HP System (server-side, UI reflects state):**
- HP ≤ 0: player moved to Hospital (Tile 30), `movedToHospital` event broadcast
- Player screen shows HP value in stat grid (initialized to 10)
- Hospital mechanic is Phase 6+ (Phase 5 stubs tile only)
